Campus Helpdesk

About the Project

This mini project centralizes student issues like complaints, lost-and-found, and quick admin communication into one clean dashboard. Built for speed, clarity, and the bare minimum drama.

3
Core Modules
~1–2 ms

100%
Mobile-friendly

Mission

Make campus problem-reporting boringly efficient. Students log issues in seconds, admins track status transparently, and no one has to spam five WhatsApp groups ever again.

  • Simple forms, strict validation, fast feedback
  • Status-based tracking to avoid “bhai dekh liya?” chaos
  • Lightweight front-end; back-end agnostic

Tech Stack

HTML5, CSS3, Vanilla JS
LocalStorage / API-ready
Modular pages

Clean, accessible UI

Modules

  • Complaints: category, details, priority, status
  • Lost & Found: item details, contact info, claim flow
  • Admin: review, update status, basic analytics

How it Works

  • Student submits form with required fields
  • Entry stored (LocalStorage now, API later)
  • Admin updates status: Open → In-Progress → Resolved
  • Student sees current status without chasing people

Roadmap

  • Auth roles: Student, Admin
  • Search and filters on submissions
  • Email/SMS notifications
  • Export CSV and minimal audit log
